Pipeline Overview

The simulation pipeline runs in Python using QuTiP. Notebooks live in the scripts/ folder of this repository. Results are exported as JSON to data/ and rendered here via Chart.js.

scripts/01_rss_finder.ipynb QuTiP status: active

Module 1 — Reduced Steady State Finder

Computes ρ_ss^{spin} for the spin-motion-bath system. Parameter scans over: detuning (ω_0 − ω_m), coupling g, motional temperature T, and Markovian noise rate γ. Outputs: RSS Bloch vector, Liouvillian spectrum (lowest 6 eigenvalues), relaxation gap Δ.

scripts/02_rss_diagnostic.ipynb QuTiP status: active

Module 2 — RSS Diagnostic Check

Initialises spin in ρ_ss^{spin}, motion in thermal state at n̄ = 5. Evolves and tracks D(ρ_spin(t), ρ_ss). Confirms RSS is a global but not local fixed point of spin dynamics. Checks Lamb-Dicke correction by scanning η.

scripts/03_mpemba_scan.ipynb QuTiP status: in progress

Module 3 — Mpemba State-Space Scan

Scans initial spin states on the Bloch sphere. For each, computes D(ρ(0), ρ_ss) and D(ρ(t*), ρ_ss) at the predicted crossing time. Maps the Mpemba manifold in state space. Identifies candidate state pairs (ρ_A, ρ_B) for the experimental test.

scripts/04_nonmarkov.ipynb QuTiP / HEOM status: planned

Module 4 — Non-Markovian Extension

Implements hierarchical equations of motion (HEOM) for a Drude-Lorentz bath spectral density. Scans bath memory time τ_bath = 1/γ. Tracks how crossing time t* scales with τ_bath. Checks Markovian limit recovery.

Figure 1 — RSS Diagnostic Check

The spin is initialised in ρ_ss^{spin}. The motional mode is initialised in a thermal state with n̄ = 5. The trace distance D(ρ_spin(t), ρ_ss) starts near zero, rises as spin-motion entanglement builds, then returns to zero at long times. This confirms that ρ_ss^{spin} is a global but not local fixed point of the spin dynamics — a prerequisite for the Mpemba experiment.

Figure 2 — Mpemba Scenario (Illustrative)

Two initial spin states: state A (farther from ρ_ss, D₀ = 0.45) and state B (closer, D₀ = 0.22). Under the same dynamics, state A crosses below state B in trace distance at t* ≈ 27. This is the Mpemba scenario. The state pair shown here is chosen for visual clarity — the scan for experimentally accessible pairs is ongoing (Module 3).

Code Sketch — RSS Finder (QuTiP)

import qutip as qt
import numpy as np

# System parameters
omega0 = 1.0    # spin frequency
omegam = 1.0    # motional mode
g      = 0.05   # spin-motion coupling
gamma  = 0.02   # Markovian decay rate
n_bar  = 5.0    # mean phonon number
N      = 20     # Fock space truncation

# Operators
sm = qt.tensor(qt.sigmam(), qt.qeye(N))      # spin lowering
sz = qt.tensor(qt.sigmaz(), qt.qeye(N))      # spin z
a  = qt.tensor(qt.qeye(2),  qt.destroy(N))   # motional lowering

# Hamiltonian
H = (omega0/2) * sz + omegam * a.dag() * a \
  + g * (sm + sm.dag()) * (a + a.dag())

# Jump operators (Markovian bath)
c_ops = [
    np.sqrt(gamma * (n_bar + 1)) * a,          # phonon decay
    np.sqrt(gamma * n_bar)         * a.dag(),   # phonon creation
]

# Solve for steady state
rho_ss_total = qt.steadystate(H, c_ops)

# Reduced steady state (spin only)
rho_ss_spin = rho_ss_total.ptrace(0)

# Trace distance function
def trace_distance(rho, sigma):
    diff = rho - sigma
    return 0.5 * (diff.dag() * diff).sqrtm().tr().real

# Evolve from RSS initialisation
rho0 = qt.tensor(rho_ss_spin, qt.thermal_dm(N, n_bar))
times = np.linspace(0, 200, 200)
result = qt.mesolve(H, rho0, times, c_ops, [])

# Compute trace distance trajectory
D_vals = [trace_distance(r.ptrace(0), rho_ss_spin)
          for r in result.states]
